Historique

... c'est un peu vieux

1978-1988

P.J. Plauger, un des développeur du langage C :

A chaque terminal, deux développeurs. Évidemment, seulement un développeur utilise le clavier, mais les autres regardent par dessus son épaule

1978-1988

P.J. Plauger : inventeur ou précurseur de la technique, sans la nommer "pair programming".

1995

Le terme arrive plus tard...

  • "Developing in pairs" pattern in Jim Complien's book
  • Constantine on peopleware, Larry Constantine

1998

Kent Beck : eXtreme programming explained

Le binômage est au centre de la pratique

2002

Laurie Williams : Pair programming illuminated

  • Premier livre dédié sur la programmation en binôme
  • Premières études empiriques sur les résultats de la programmation en paire

Programmation en binôme

Définition : une vieille méthode de développement Agile dans laquelle deux développeurs travaillent ensemble sur un même poste de travail.

Pour 15% d'augmentation du temps de développement

  • diminue les coûts de productions
  • améliore satisfaction des développeurs
  • améliore la qualité du design
  • permet de faire de la revue de code gratuitement
  • améliore les capacités de résolution de problème
  • améliore la cohésion d'équipe et la communication
  • améliore les problématiques de recrutement

C'est tu pas beautiful mon buddy boy ?

Avantages pour un développeur

  • rend les tâches monotones plus intéressantes
  • rend les tâches difficiles plus faciles
  • permet de découvrir de nouveaux idiomes

Avantages pour une société

  • augmente le "nombre bus"
  • diminue les risques du projet
  • les inspections sont effectuées JIT

Avantages pour un produit

  • meilleurs design, parce que plus de solutions envisagées
  • même problème, moins de code
  • moins de chance de faire la solution "hacky"
